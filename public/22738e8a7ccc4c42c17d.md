\
\*\*GitHub（ギットハブ）\*\*は、プログラムのソースコードを管理するツール「**Git**」を利用した、世界最大のホスティングサービスです。個人開発からオープンソースプロジェクト、企業の業務開発まで幅広く利用されています。

## 1. Git と GitHub の違い

よく混同されますが、役割が異なります。

* **Git:** コンピュータ上でファイルの変更履歴を記録する「ツール（ソフトウェア）」そのもの。
* **GitHub:** インターネット上でGitのリポジトリ（保管庫）を預かり、開発者同士の交流や連携を助ける「Webサービス」。


---

## 2. GitHub の主要な機能

GitHubが「単なるストレージ」ではない理由は、以下の強力な機能にあります。

| 機能名 | 役割  |
|-----|-----|
| **Push / Pull** | 自分のPCで書いたコードをGitHubに送り（Push）、最新版を取り込む（Pull）。 |
| **Pull Request (プルリク)** | 自分が修正したコードを「本体に取り込んでください」と依頼する機能。レビューの要。 |
| **Issues** | バグ報告、機能要望、タスク管理などを掲示板形式で管理。 |
| **GitHub Actions** | コードを保存した瞬間に、自動でテストやサーバーへの公開（デプロイ）を行う自動化機能。 |
| **GitHub Pages** | HTMLファイルを置くだけで、無料でWebサイトを公開できる機能。 |


---

## 3. GitHub を使うメリット


1. **ポートフォリオになる:** 自分が書いたコードを公開しておくことで、エンジニアとしての実力を証明する「履歴書」代わりになります。
2. **過去に戻れる:** 「昨日の状態に戻したい」といった変更履歴の管理が完璧にできます。
3. **世界中のコードが読める:** 有名なオープンソース（VS CodeやLinuxカーネルの一部など）のソースコードを読み、学ぶことができます。
4. **共同開発の円滑化:** 誰が、いつ、どこを、なぜ書き換えたのかがすべて可視化されます。


---

## 4. SSH接続とGitHub（これまでのWikiとの関連）

GitHubに自分のコードを送信（Push）する際、**SSH接続**を利用するのが一般的かつ安全です。

* 自分のPCで作成した「公開鍵」をGitHubの設定画面に登録します。
* これにより、パスワードを毎回入力することなく、安全に通信が行えます。
* **VS Code**を使えば、ボタン一つでGitHubへの保存が完結します。


---

## 5. 2026年のトレンド：GitHub Copilot

現代のGitHubで最も注目されているのが、AIによるコーディング支援「**GitHub Copilot**」です。

* **AIがコードを書く:** 関数名やコメントを書くだけで、AIが続きのコードを提案します。
* **チャット形式の相談:** エラーが出た際、エディタ上でAIに修正案を聞くことができます。
* **効率化:** プログラミング作業の時間が従来の半分以下になるとも言われています。


---

## 6. 基本用語集

* **Repository (リポジトリ):** プロジェクトごとの保管箱。
* **Commit (コミット):** 変更内容を「記録」すること。
* **Branch (ブランチ):** 開発の流れを枝分かれさせること（新機能用、バグ修正用など）。
* **Merge (マージ):** 枝分かれした開発内容を一つに統合すること。
