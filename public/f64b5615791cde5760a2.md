# k8sでnginxを立ててサイトを公開するまで

(MetalLB + Traefik + cert-manager + nginx)

## 1. 概要

本ドキュメントでは、ベアメタル環境のKubernetesにおいて、外部IPの割り当てからHTTPS化されたアプリケーション公開までの手順を記述します。

### スタック構成

* **L4 LoadBalancer**: MetalLB
* **Ingress Controller**: Traefik
* **Certificate Management**: cert-manager (Let's Encrypt)
* **Sample Application**: nginx


---

## 2. 構築手順

### Step 1: MetalLB のインストール

ネットワーク内の空きIPアドレスをK8sサービスに割り当て可能にします。


1. **インストール**:

```bash
kubectl apply -f https://raw.githubusercontent.com/metallb/metallb/v0.14.3/config/manifests/metallb-native.yaml
```


2. **IPアドレスプールの設定**: `IPAddressPool` と `L2Advertisement` を作成し、割り当てるIPレンジ（例: `192.168.1.200-192.168.1.210`）を指定します。

### Step 2: Traefik のインストール

Ingress（L7）として動作し、外部からのリクエストを各Serviceにルーティングします。


1. **Helmでのインストール推奨**:

```bash
helm repo add traefik https://traefik.github.io/charts
helm install traefik traefik/traefik
```


2. **確認**: `kubectl get svc` で Traefik の `EXTERNAL-IP` に MetalLB から払い出されたIPが表示されていることを確認します。

### Step 3: cert-manager の設定 (CertResolver)

SSL/TLS証明書の自動発行を管理します。


1. **インストール**:

```bash
kubectl apply -f https://github.com/cert-manager/cert-manager/releases/download/v1.13.0/cert-manager.yaml
```


2. **ClusterIssuer の作成**: Let's Encrypt等と連携するための設定ファイルを作成します。メールアドレスとチャレンジ方式（HTTP-01 or DNS-01）を定義します。

### Step 4: nginx アプリケーションの公開

最後にテスト用アプリをデプロイし、外部ドメインからアクセス可能にします。


1. **Deployment & Service**: nginxを起動。
2. **Ingress (または IngressRoute)**:

* ホスト名（例: `nginx.example.com`）を指定。
* `cert-manager.io/cluster-issuer` アノテーションを付与して自動証明書発行を有効化。


---

## 3. トラブルシューティング

| 現象  | 確認項目 |
|-----|------|
| `EXTERNAL-IP` が `<pending>` のまま | MetalLBのIPアドレスプールがネットワーク帯域と重複していないか、ARPが通るか確認。 |
| 証明書が `Ready: False` | `kubectl describe certificate` でエラーを確認。80番ポートが外部から到達可能か（HTTP-01の場合）。 |
| 404 Not Found | Traefikのダッシュボードでルールのマッチング条件を確認。 |
